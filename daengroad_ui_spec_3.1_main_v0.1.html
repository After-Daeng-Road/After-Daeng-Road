1 ■ 메인 검색 영역

1-1 시간 슬라이더
  • 범위 1h ~ 6h, 30분 단위 눈금
  • 디폴트 3h · 사용자 마지막 선택값 쿠키 저장
  • 슬라이더 변경 시 API 호출 ✗ (클라이언트 state만)
  • 키보드 ←/→ : 30분씩 이동 · 모바일 터치 드래그
1-2 출발지 입력
  • 입력 모드 3종: GPS · 주소 · 우편번호
  • "현 위치" 버튼 → navigator.geolocation.getCurrentPosition() + 권한 동의 모달
  • 주소 자동완성 → GET kakao.local.keyword/address (디바운스 300ms)
  • 좌표 확정 시 client state에 저장 · DB 저장 ✗ (개인정보 최소화 원칙)
  • 실패 시 inline 에러 메시지 + 수동 입력 권유
1-3 펫 선택 드롭다운
  • 페이지 진입 시 GET /api/pets → 사용자 등록 펫 목록 캐싱
  • 다중 펫 등록자: 1순위 펫 디폴트 · 변경 시 client state만 갱신
  • 펫 미등록: "펫 등록하기" CTA 노출 + 추천받기 버튼 비활성
1-4 출발 시각 선택
  • "지금" 디폴트(서버 시각 자동) · 30분 단위로 +30분 ~ +6h 미래 선택
  • 한적도 점수가 시간대별 가변이므로 필수 입력 (NULL 불가)
1-5 "지금 추천받기" 버튼 ★ 핵심 액션
  • 미로그인 시: 카카오 로그인 모달 호출 → 로그인 후 입력값 보존하여 자동 재요청
  • 클릭 → POST /api/recommendations.search
  • Body: { pet_id, departure: {lat,lng}, time_h, start_at }
  • 내부 처리: Edge Function time-slider-recommender 호출
    • ① TourAPI4 locationBasedList2 + 펫 locationBasedList (반경 후보)
    • ② 카카오모빌리티 ETA (24h KV 캐시)
    • ③ 데이터랩 locgoVisitorList + poi_forecasts (현재·30일 한적도)
    • ④ 펫 detailPetTour + UGC 검증 결합
    • ⑤ 점수 정렬 → 상위 3개 + reason_chips
  • 응답: { recommendation_id, items: [3 POI + reason_chips] }
  • 로딩: 스켈레톤 카드 3개 (목표 ≤ 5s)
  • 실패: 토스트 + 재시도 · Sentry 자동 리포트

2 ■ 추천 결과 영역

2-1 POI 카드 (상위 3개)
  • 이미지 1장 (TourAPI detailImage2 첫 번째)
  • 이름·주소·소스 라벨(두루누비 / 펫동반)
  • 카드 클릭 → GET /api/pois/:id + 페이지 이동 /poi/:id
2-2 3줄 근거 칩 ★ 차별점
  • 거리: ETA 거리·소요시간 (카카오모빌리티)
  • 한적도: 0~100점 (데이터랩 + UGC 결합)
  • 검증: 펫동반 검증 배지 + 방문자 N명
  • 각 칩 클릭 → 데이터 출처 모달(투명성) — 기획 강점 어필
2-3 30일 예측 라벨 (파란 톤)
  • "내일 같은 시간 N점 · 이번 주 평균 N점"
  • 출처: poi_forecasts 테이블 (관광지 집중률 예측 OpenAPI)
  • 표본 부족 시 "표본 부족" 라벨로 대체
2-4 액션 버튼
  • 길찾기 → 카카오모빌리티 deep link (앱 미설치 시 카카오맵 웹 fallback)
  • 저장 → POST /api/saved-pois (미로그인 시 로그인 모달)
  • 공유 → 카카오톡 공유 SDK (코스 URL + 미리보기 카드)

3 ■ 이메일 알림 CTA

3 알림 미설정 사용자만 노출
  • 조건: users.email_notify_enabled === false
  • "이메일 알림 켜기" 클릭 → PATCH /api/users/me/notify-settings
  • Body: { enabled: true, time: '18:00', days: ['MON','TUE','WED','THU','FRI'] }
  • 닫기 시 1일 미노출 (쿠키 저장)

4 ■ 검증 배지 가이드 플로팅

4 플로팅 배지 (PC 우하단 / 모바일 우하단)
  • 클릭 → 검증 배지 시스템 설명 모달 (3명 방문 + EXIF 사진 + 6개월)
  • 스크롤 시 함께 이동 · "추천 결과" 영역에서는 z-index 낮춤
  • 닫기 시 1일 미노출 (쿠키)

● 페이지 진입 초기 호출 시퀀스

  1. GET /api/auth/session — 로그인 확인 (Auth.js + Supabase)
  2. 로그인 시 병렬 호출:
    • GET /api/users/me — 알림 설정·기본 출발지·언어
    • GET /api/pets — 등록 펫 목록
  3. 미로그인 시: 헤더 우상단 "로그인" 버튼 표시 · 추천받기 클릭 시 카카오 로그인 모달

● 상태 / 에러 핸들링

  • 로딩: 추천 카드 자리에 스켈레톤 3개 (Tailwind animate-pulse)
  • 빈 결과: "조건에 맞는 한적한 곳을 찾지 못했어요" + 시간 늘리기·반경 확장 제안
  • 서버 에러: 토스트 + 재시도 버튼 · Sentry 자동 리포트
  • 429 Rate Limit: "잠시 후 다시 시도" + 30초 카운트다운
  • 미로그인 추천 클릭: 입력값 sessionStorage 보존 → 로그인 후 자동 재요청
